# Requirements: 罐装机数字孪生 Phase 2 — 3D 模型孪生体

> **目标**：用真实 3D 模型替换图片方案，实现部件级交互、告警高亮、历史趋势，构建真正的数字孪生体验  
> **优先级**：P0  
> **预估工作量**：15–20 天（模型到位后）  
> **前置条件**：设备厂商交付 glTF/GLB 或 STEP 模型文件

## 1. 背景与动机

### 1.1 当前问题

- Phase 1 使用 14 张图片做伪 3D，旋转时有跳切感，不是真正的 3D 渲染。
- 无法点击设备部件查看局部参数，所有数据只能在侧边面板展示。
- 告警只能在文字列表中显示，无法在 3D 场景中直观定位故障部位。
- 没有历史趋势，无法回溯设备运行状况。
- 以上限制导致当前系统只是「带图片的数据看板」，称不上「数字孪生」。

### 1.2 设计目标

- 用 glTF 3D 模型替换图片方案，实现真正的实时 3D 渲染。
- 支持部件级交互：鼠标悬停高亮、点击弹出部件参数面板。
- 告警时对应部件在 3D 场景中闪烁/变色，一眼定位故障位置。
- 增加历史趋势图表，支持按时间范围查看测点变化曲线。
- 保持 Phase 1 的所有功能（数据面板、Mock 引擎、Docker 部署）不退化。

## 2. 用户故事

### 2.1 作为管理者——3D 模型查看

**故事 1**：真实 3D 模型交互
```
作为 生产管理者
我希望 在浏览器中看到罐装机的真实 3D 模型，可以自由旋转、缩放、平移
这样我可以 从任意角度查看设备，比图片方案更直观、更专业
```

**验收标准**：
- [ ] glTF/GLB 模型正确加载并渲染，外观与实际设备一致
- [ ] 鼠标拖拽自由旋转，滚轮缩放，右键平移
- [ ] 模型有合理的光照和材质，不锈钢有金属质感
- [ ] 首次加载 < 5 秒（模型文件 < 50 MB）

### 2.2 作为管理者——部件级交互

**故事 2**：点击部件查看参数
```
作为 生产管理者
我希望 鼠标悬停在灌装头上时高亮显示，点击后弹出该部件的运行参数
这样我可以 快速定位到关心的部件，查看其具体状态
```

**验收标准**：
- [ ] 鼠标悬停在可交互部件上时，部件边缘高亮（outline 或发光效果）
- [ ] 鼠标变为手型指针，提示可点击
- [ ] 点击部件后弹出浮动面板，显示该部件关联的测点数据
- [ ] 点击空白处或关闭按钮可收起浮动面板
- [ ] 至少支持以下部件的交互：灌装头组件、传送带、液缸、控制柜

### 2.3 作为管理者——告警 3D 高亮

**故事 3**：告警部位在 3D 上闪烁
```
作为 生产管理者
我希望 设备发生告警时，故障部位在 3D 模型上闪烁变色
这样我可以 一眼看到哪个部位出了问题，不用逐条读告警列表
```

**验收标准**：
- [ ] critical 告警：对应部件闪烁红色（0.5 秒周期）
- [ ] warning 告警：对应部件持续显示橙色
- [ ] 多个部件同时告警时各自独立高亮
- [ ] 告警恢复后部件颜色自动恢复正常

### 2.4 作为管理者——历史趋势

**故事 4**：查看测点历史曲线
```
作为 生产管理者
我希望 选择一个测点后查看其过去一段时间的变化曲线
这样我可以 分析设备运行趋势，发现潜在问题
```

**验收标准**：
- [ ] 可选择任意测点查看历史曲线
- [ ] 支持时间范围选择：最近 1 小时 / 4 小时 / 8 小时 / 24 小时
- [ ] 曲线图显示值、时间轴、正常范围区间（绿色背景带）
- [ ] 超限区间用红色标注

## 3. 功能需求

### 3.1 3D 模型渲染

#### FR-1：glTF 模型加载与渲染

**需求**：加载 glTF/GLB 格式的罐装机 3D 模型，在浏览器中实时渲染。

**验收标准**：
- [ ] 使用 `useGLTF` (Drei) 加载模型
- [ ] 模型正确显示，部件层级结构保留
- [ ] 光照合理：环境光 + 方向光 + 可选 HDRI 环境贴图
- [ ] 材质正确：金属/非金属区分，玻璃部件半透明
- [ ] 支持模型文件热替换（换模型只需替换文件，不改代码）

#### FR-2：相机控制升级

**需求**：从 Phase 1 的图片旋转升级为真正的 3D 相机控制。

**验收标准**：
- [ ] 左键拖拽旋转
- [ ] 右键拖拽平移
- [ ] 滚轮缩放
- [ ] 双击部件自动聚焦（相机平滑移动到该部件正前方）
- [ ] 预设视角按钮：正面、背面、左侧、右侧、俯视

### 3.2 部件交互

#### FR-3：部件悬停高亮

**需求**：鼠标悬停在可交互部件上时显示高亮效果。

**验收标准**：
- [ ] Raycaster 检测鼠标悬停的部件
- [ ] 高亮效果：outline 描边（白色/蓝色）或发光效果
- [ ] 鼠标变为 `pointer` 手型
- [ ] 悬停时显示部件名称 tooltip

#### FR-4：部件点击 → 参数面板

**需求**：点击部件弹出浮动面板，显示该部件关联的测点数据。

**验收标准**：
- [ ] 点击部件后在 3D 场景上方弹出浮动面板
- [ ] 面板显示：部件名称、关联测点列表（名称+值+单位）、部件状态
- [ ] 面板位置跟随部件在屏幕上的投影位置
- [ ] 点击空白处或 X 按钮关闭面板

#### FR-5：部件 → 测点映射配置

**需求**：部件名称与测点 ID 的映射关系可配置，不硬编码。

**验收标准**：
- [ ] 映射关系定义在配置文件中（JSON/TypeScript）
- [ ] 配置格式：`{ partName: string, label: string, pointIds: string[] }`
- [ ] 新增部件或测点只需修改配置，不改组件代码

### 3.3 告警 3D 高亮

#### FR-6：告警部件变色/闪烁

**需求**：活跃告警关联的部件在 3D 模型上变色或闪烁。

**验收标准**：
- [ ] 根据告警的 `point_id` 查找映射配置，定位到对应部件
- [ ] critical：部件材质变红 + 0.5 秒闪烁动画
- [ ] warning：部件材质变橙，不闪烁
- [ ] 多告警叠加时取最高等级的效果
- [ ] 告警恢复后材质自动还原

### 3.4 历史趋势

#### FR-7：测点历史数据存储

**需求**：后端将测点值按时间序列存储，支持按时间范围查询。

**验收标准**：
- [ ] Mock 引擎每次 tick 将测点值追加到内存环形缓冲区
- [ ] 保留最近 24 小时的数据（按秒粒度）
- [ ] API：`GET /api/twin/equipment/{id}/history?point_id=speed&hours=4`

#### FR-8：历史趋势图表

**需求**：前端展示测点历史曲线。

**验收标准**：
- [ ] 用 Recharts 的 LineChart 渲染
- [ ] X 轴：时间；Y 轴：测点值
- [ ] 正常范围用绿色半透明背景带标注
- [ ] 超限区间用红色标注
- [ ] 支持时间范围切换：1h / 4h / 8h / 24h

- 模型加载后渲染帧率 >= 30fps
- 模型文件 < 50 MB（GLB 格式）

**验收标准**：
- [ ] Chrome DevTools Performance 面板确认 >= 30fps
- [ ] 模型加载时显示进度条

### 4.2 兼容性

**NFR-2：Phase 1 功能不退化**
- Phase 1 的所有面板（状态、产量、参数、OEE、告警）继续正常工作

**验收标准**：
- [ ] Phase 1 验收清单中的所有项仍然通过

### 4.3 可配置性

**NFR-3：模型可替换**
- 更换设备型号时，只需替换 GLB 文件 + 部件映射配置

**验收标准**：
- [ ] 模型路径通过配置指定，不硬编码
- [ ] 部件映射通过配置文件定义

## 6. 约束与假设

### 6.1 约束
- 模型文件由设备厂商提供，格式和质量不完全可控
- 如果厂商提供 STEP 格式，需要额外的转换步骤（Blender）
- 部件拆分粒度取决于厂商提供的模型结构

### 6.2 假设
- 厂商能在 2-4 周内交付模型文件
- 模型按功能部件分组命名（至少灌装头、传送带、控制柜独立）
- 模型面数在 200 万以内，GLB 文件 < 50 MB

## 8. 风险与缓解

### 8.1 风险：厂商交付延迟

**影响**：Phase 2 无法启动

**缓解**：
- 并行方案：用开源罐装机/工业设备模型做技术验证
- 催促方案：`docs/3D_MODEL_REQUEST.md` 已准备好，明确格式和要求
- 备选方案：用 AI 工具（Meshy/Tripo3D）从图片生成粗略 3D 模型

### 8.2 风险：模型未按部件拆分

**影响**：无法实现部件交互和告警高亮

**缓解**：
- 在 Blender 中手动拆分部件（增加 1-2 周工作量）
- `3D_MODEL_REQUEST.md` 中已明确要求部件拆分

### 8.3 风险：模型面数过高导致浏览器卡顿

**影响**：渲染帧率 < 30fps

**缓解**：
- 在 Blender 中做 Decimate（减面）处理
- 使用 Three.js LOD（多层次细节）
- 远距离用简化模型，近距离用高精度模型
