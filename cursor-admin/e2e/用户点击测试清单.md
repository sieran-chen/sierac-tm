# 用户点击测试清单（结合 Spec E2E 要求）

**说明**：用浏览器打开管理端后，按下列步骤逐项点击并打勾。  
**管理端**：http://8.130.50.168:3000（或本地 http://localhost:3000）

---

## 一、立项与治理（对应 cursor-admin-projects E2E_VERIFICATION）

| 步骤 | 操作 | 预期 | 通过 |
|------|------|------|------|
| 1 | 侧栏点击 **项目** | 进入项目管理页，见「项目管理」标题、**刷新**、**新建项目** 按钮 | ☐ |
| 2 | 点击 **新建项目** | 弹窗打开：项目名称、描述、工作目录规则、参与成员、**仓库创建方式**（关联已有仓库 / 自动创建 GitLab / 自动创建 GitHub）、创建人邮箱 | ☐ |
| 3a | 选择 **关联已有仓库** | 出现说明「保存后，在工作目录符合规则的 Cursor 会话将自动归属…」及 **已有仓库地址 \*** 输入框（必填） | ☐ |
| 3b | 填写 项目名称、工作目录规则、创建人邮箱、**已有仓库地址**（如 `https://github.com/org/repo.git`） | **保存** 可点击 | ☐ |
| 3c | 点击 **保存** | 弹窗关闭，顶部出现绿色提示「项目已创建。若未看到新项目，请点击刷新。」；列表刷新，可点 **刷新** 确认 | ☐ |
| 4 | 侧栏点击 **项目参与** | 进入项目参与页，见「按项目与成员汇总」、未归属说明、汇总表或空状态 | ☐ |
| 5 | 侧栏点击 **我的项目** | 进入我的项目页，见成员选择与说明 | ☐ |
| 6 | 返回 **项目**，点击列表中某项目名（如 sierac-tm） | 进入项目详情页：成本面板、参与面板、贡献面板，有「返回项目列表」 | ☐ |

## 自动运行（Playwright）

在**能访问部署地址**的机器上（或本地先 `cd cursor-admin/web && npm run dev` 后用 `http://localhost:5173`）：

```bash
cd cursor-admin/e2e
npm install
npx playwright install chromium
E2E_BASE_URL=http://8.130.50.168:3000 npm run e2e
```

有界面模式（可见浏览器窗口）：

```bash
E2E_BASE_URL=http://8.130.50.168:3000 npx playwright test --headed
```

若本机无法访问 8.130.50.168，可在服务器或能访问该地址的环境执行上述命令。
