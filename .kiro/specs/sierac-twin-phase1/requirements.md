# Requirements: 罐装机数字孪生 Phase 1

> **目标**：基于现有多角度图片实现可交互 3D 设备查看器 + 模拟运行数据面板，验证数字孪生技术路线  
> **优先级**：P1  
> **预估工作量**：20–25 天（1 人全栈）

## 1. 背景与动机

### 1.1 当前问题

- 罐装机设备运行状态只能在现场查看，管理层无法远程掌握设备状况。
- 设备数据分散在 PLC/SCADA 系统中，缺乏统一可视化入口。
- 缺少直观的设备 3D 展示手段，向客户/管理层汇报时只能用照片或文字描述。

### 1.2 设计目标

- 提供可交互的 3D 设备查看器，支持鼠标拖拽旋转、缩放，直观展示罐装机外观。
- 在 3D 场景旁展示设备运行数据面板（状态、产量、参数、告警、OEE）。
- 定义清晰的数据接口抽象，Phase 1 用 Mock 数据，后续可无缝切换真实数据源。
- 独立子项目，不与 Sierac-tm 激励平台耦合。

## 2. 用户故事

### 2.1 作为管理者——远程查看设备

**故事 1**：查看设备 3D 外观
```
作为 生产管理者
我希望 在浏览器中看到罐装机的 3D 展示，可以拖拽旋转查看各个角度
这样我可以 直观了解设备外观，向客户或上级汇报时有可视化素材
```

**验收标准**：
- [ ] 页面打开 2 秒内显示罐装机图片
- [ ] 鼠标拖拽可 360° 水平旋转查看
- [ ] 支持上下俯仰角度调整（有范围限制）
- [ ] 滚轮缩放，有最大/最小限制
- [ ] 旋转时图片切换平滑，无明显卡顿或跳切

**故事 2**：查看设备运行状态
```
作为 生产管理者
我希望 在 3D 场景旁看到设备的实时运行数据
这样我可以 远程掌握设备运行状况，及时发现异常
```

**验收标准**：
- [ ] 面板实时显示设备运行状态（运行/待机/故障/维护），状态灯颜色正确
- [ ] 显示当日产量、目标、达成率进度条
- [ ] 显示关键参数（速度、温度、压力、OEE），参数超限时变色提示
- [ ] 显示告警列表，按等级排序，critical 排最前
- [ ] 数据每 5 秒自动更新，无需手动刷新

### 2.2 作为演示者——向管理层展示

**故事 3**：一键启动演示
```
作为 技术负责人
我希望 用 docker compose up 一键启动整个系统
这样我可以 快速搭建演示环境，无需复杂配置
```

**验收标准**：
- [ ] `docker compose up` 一键启动前后端
- [ ] 浏览器访问即可看到完整界面
- [ ] Mock 数据自动运行，无需额外配置

## 3. 功能需求

### 3.1 3D 查看器

#### FR-1：多角度图片展示

**需求**：基于 14 张多角度图片，在浏览器中渲染可交互的设备 3D 查看器。

**验收标准**：
- [ ] 加载 14 张图片（前/后/左/右/上/下 + 8 个斜角）
- [ ] 图片按球面坐标映射，相机旋转时选择最近视角的图片显示
- [ ] 图片切换时有 crossfade 过渡动画（200ms）

#### FR-2：相机交互控制

**需求**：支持鼠标拖拽旋转、滚轮缩放，有合理的角度和距离限制。

**验收标准**：
- [ ] 水平旋转 360° 无限制
- [ ] 垂直俯仰限制在 -60° 到 +60°
- [ ] 缩放距离限制在 2x 到 0.5x 范围
- [ ] 操作响应流畅（60fps）

#### FR-3：视角指示器

**需求**：在 3D 查看器角落显示当前视角方向的小指示器。

**验收标准**：
- [ ] 显示当前相机朝向（如「前方」「右后方」）
- [ ] 随拖拽实时更新

### 3.2 数据面板

#### FR-4：设备状态卡片

**需求**：显示设备当前运行状态，用颜色区分。

**验收标准**：
- [ ] 显示状态文字 + 状态指示灯
- [ ] 运行=绿、待机=蓝、故障=红、维护=橙
- [ ] 显示当前状态持续时长

#### FR-5：产量面板

**需求**：显示当日产量、目标、达成率。

**验收标准**：
- [ ] 显示当日产量数值（带千分位格式化）
- [ ] 显示当日目标数值
- [ ] 进度条展示达成率百分比
- [ ] 达成率 < 50% 时进度条变色警示

#### FR-6：关键参数面板

**需求**：以表格形式展示关键运行参数。

**验收标准**：
- [ ] 至少展示：灌装速度、物料温度、灌装压力、活跃灌装头数
- [ ] 每项显示名称、当前值、单位
- [ ] 值超出正常范围时文字变红 + 背景浅红

#### FR-7：OEE 仪表盘

**需求**：用环形仪表盘展示设备综合效率。

**验收标准**：
- [ ] 显示 OEE 总值（百分比）
- [ ] 可展开查看可用率、性能率、良品率分解

#### FR-8：告警面板

**需求**：展示当前活跃告警列表。

**验收标准**：
- [ ] 按等级排序（critical > warning > info）
- [ ] 每条告警显示等级图标、消息、时间
- [ ] critical 告警红底白字，warning 橙底，info 蓝底

### 3.3 数据层

#### FR-9：Mock 数据引擎

**需求**：后端提供状态机驱动的模拟数据，模拟真实设备运行节奏。

**验收标准**：
- [ ] 设备有状态机（运行→待机→故障→维护），状态转移概率合理
- [ ] 连续量（速度、温度、压力）渐变而非跳变
- [ ] 故障时速度降为 0、告警自动触发
- [ ] 模拟三班制运行节奏

#### FR-10：REST API

**需求**：后端提供标准 REST 接口，前端轮询获取数据。

**验收标准**：
- [ ] `GET /api/twin/equipment/{id}/summary` 返回设备摘要
- [ ] `GET /api/twin/equipment/{id}/telemetry` 返回全部测点实时值
- [ ] `GET /api/twin/equipment/{id}/alarms?active=true` 返回活跃告警
- [ ] 响应时间 < 100ms

## 4. 非功能需求

### 4.1 性能

**NFR-1：页面加载性能**
- 首屏加载时间 < 3 秒（本地网络）
- 14 张图片总计约 760 KB，需预加载

**验收标准**：
- [ ] 首屏 3 秒内可交互
- [ ] 图片预加载完成后旋转无延迟

### 4.2 兼容性

**NFR-2：浏览器兼容**
- 支持 Chrome 100+、Edge 100+（工厂环境主流浏览器）

**验收标准**：
- [ ] Chrome 和 Edge 最新版本下功能正常

### 4.3 响应式

**NFR-3：屏幕适配**
- 支持 1920×1080 和 1366×768 两种主流分辨率

**验收标准**：
- [ ] 1920px 宽度下左右分栏布局正常
- [ ] 1366px 宽度下布局不溢出、不重叠

### 4.4 部署

**NFR-4：Docker 化部署**
- 前后端均可 Docker 化，docker compose 一键启动

**验收标准**：
- [ ] `docker compose up` 启动后浏览器可访问
- [ ] 无需额外手动配置

## 6. 约束与假设

### 6.1 约束
- 仅使用现有 14 张 PNG 图片，不新增拍摄
- Phase 1 不接入真实 PLC/SCADA 数据
- 独立子项目，不修改 Sierac-tm 现有代码

### 6.2 假设
- 14 张图片覆盖的角度足以提供可接受的旋转体验
- Mock 数据足以验证技术路线和 UI 效果
- 工厂侧后续可提供 OPC-UA 或 MQTT 数据接口（Phase 2 前提）

## 8. 风险与缓解

### 8.1 风险：14 张图片旋转效果不够平滑

**影响**：用户体验差，演示效果不佳

**缓解**：
- 降低旋转灵敏度，减少切换频率
- 增大 crossfade 时间（200ms → 400ms）
- 后续可用 AI 生成中间角度图片补充

### 8.2 风险：Three.js 生态学习成本

**影响**：开发周期延长

**缓解**：
- React Three Fiber 降低了 Three.js 的使用门槛
- Drei 提供了 OrbitControls 等开箱即用组件
- Phase 1 不涉及复杂 3D 渲染（无模型、无光照、无物理）
