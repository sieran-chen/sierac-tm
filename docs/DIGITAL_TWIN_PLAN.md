## 二、项目定位与边界

### 2.1 本项目做什么

1. **可交互 3D 设备查看器**：基于多角度图片，支持鼠标拖拽旋转、缩放，直观展示罐装机外观
2. **运行数据面板**：在 3D 场景旁展示设备运行状态、产量、关键参数、告警
3. **数据接口抽象**：定义清晰的设备-测点-实时值接口，Mock 与真实数据源可无缝切换

### 2.2 本项目不做什么

| 不做                 | 原因                    |
| ------------------ | --------------------- |
| 反向控制设备             | 安全风险高，不在数字孪生展示范畴      |
| 与 Sierac-tm 激励系统耦合 | 业务域完全不同，独立部署          |
| 与 sierac-mes 强耦合   | 未来可通过设备 ID 松耦合对接      |
| 多设备/多产线管理          | Phase 1 聚焦单台设备，验证技术路线 |

### 2.3 代码组织

**独立子目录 `sierac-twin/`**，位于本仓库根目录下，独立 `package.json`，独立构建与部署。

理由：

- 技术栈与 `cursor-admin/web/` 有差异（需引入 Three.js 生态）
- 部署节奏不同（数字孪生可能部署在工厂内网）
- 避免污染现有 Sierac-tm 项目的依赖树

### 3.3 3D 展示技术方案

**核心问题**：只有 14 张 2D 图片，没有 3D 模型。

**推荐方案：球面投影 + 视角插值**

将 14 张图片映射到球面坐标系，用户拖拽旋转相机时，根据相机角度选择最接近的 2-4 张图片进行加权混合（crossfade），实现平滑的「伪 3D」旋转效果。

```
相机角度 → 计算与每张图片视角的角度距离 → 取最近的图片显示
                                          → 旋转过渡时做 opacity crossfade
```

具体实现：

| 步骤        | 内容                                                               |
| --------- | ---------------------------------------------------------------- |
| 1. 图片视角标注 | 为 14 张图标注球面坐标 (theta, phi)：前=0°, 右=90°, 后=180°, 左=270°, 上/下=±90° |
| 2. 相机控制   | OrbitControls 限制为仅旋转（禁止平移），限制俯仰角范围                               |
| 3. 视角匹配   | 每帧计算相机方向与 14 个视角的角度距离，选最近的 1-2 张                                 |
| 4. 过渡动画   | 切换图片时做 200ms opacity crossfade，避免生硬跳切                            |
| 5. 渲染载体   | 用 R3F 的 `<Billboard>` 或 `<Plane>` 承载图片，始终面向相机                    |

**备选方案（更简单但效果差）**：立方体六面贴图，只用前/后/左/右/上/下 6 张，斜角图片浪费。不推荐。

**升级路径**：有 glTF 模型后，替换为 `<primitive object={gltf.scene} />`，相机控制和数据面板代码不变。

## 四、数据模型

### 4.1 设备（Equipment）

```typescript
interface Equipment {
  id: string;              // 唯一标识，如 "filler-001"
  name: string;            // 显示名称，如 "1号罐装机"
  model: string;           // 设备型号
  location: string;        // 安装位置，如 "A车间-1号线"
  status: EquipmentStatus; // 运行状态
  imageSet: string;        // 3D 图片集路径
}

type EquipmentStatus = "running" | "idle" | "fault" | "maintenance";
```

### 4.2 测点（DataPoint）

```typescript
interface DataPoint {
  id: string;              // 测点 ID，如 "speed"
  name: string;            // 显示名称，如 "线速"
  type: "number" | "enum" | "boolean";
  unit?: string;           // 单位，如 "瓶/分钟"
  min?: number;            // 正常范围下限
  max?: number;            // 正常范围上限
  alarmThreshold?: {       // 告警阈值
    warning: number;
    critical: number;
  };
}
```

### 4.3 实时值（TelemetryValue）

```typescript
interface TelemetryValue {
  pointId: string;         // 关联测点 ID
  value: number | string | boolean;
  timestamp: string;       // ISO 8601
  quality: "good" | "bad" | "uncertain";
}
```

### 4.4 告警（Alarm）

```typescript
interface Alarm {
  id: string;
  equipmentId: string;
  pointId?: string;        // 关联测点（可选）
  level: "info" | "warning" | "critical";
  message: string;
  startTime: string;
  endTime?: string;        // null = 未恢复
  acknowledged: boolean;
}
```

### 4.5 罐装机具体测点清单（建议最小集）

基于罐装机通用参数，Phase 1 模拟以下测点：

| 测点 ID           | 名称     | 类型     | 单位   | 正常范围       | 说明                             |
| --------------- | ------ | ------ | ---- | ---------- | ------------------------------ |
| `status`        | 运行状态   | enum   | -    | -          | running/idle/fault/maintenance |
| `speed`         | 灌装速度   | number | 瓶/分钟 | 200-600    | 当前线速                           |
| `today_output`  | 当日产量   | number | 瓶    | 0-50000    | 累计产量                           |
| `today_target`  | 当日目标   | number | 瓶    | -          | 计划产量                           |
| `fill_volume`   | 灌装容量   | number | mL   | 330-500    | 当前灌装规格                         |
| `fill_accuracy` | 灌装精度   | number | %    | 99.0-100.0 | 灌装量合格率                         |
| `temperature`   | 物料温度   | number | °C   | 2-8        | 灌装物料温度                         |
| `pressure`      | 灌装压力   | number | bar  | 2.5-4.0    | CO₂ 背压                         |
| `head_active`   | 活跃灌装头  | number | 个    | 24-36      | 正在工作的灌装头数                      |
| `head_total`    | 灌装头总数  | number | 个    | 36         | 设备灌装头总数                        |
| `reject_count`  | 剔除数    | number | 瓶    | 0-100      | 当日剔除瓶数                         |
| `runtime_today` | 当日运行时长 | number | 小时   | 0-24       | 累计运行                           |
| `oee`           | 设备综合效率 | number | %    | 70-95      | OEE = 可用率×性能率×良品率              |

## 八、Mock 数据引擎设计

Phase 1 的 Mock 引擎需要生成**逼真的**模拟数据，而非纯随机数。

### 8.1 设计原则

| 原则     | 说明                              |
| ------ | ------------------------------- |
| 状态驱动   | 设备有状态机（运行→待机→故障→维护），不同状态下参数范围不同 |
| 渐变而非跳变 | 速度、温度等连续量在相邻采样点之间变化幅度有限         |
| 因果关联   | 故障时速度降为 0、告警触发；维护时产量停止累加        |
| 班次模拟   | 模拟三班制（8h 一班），换班时有短暂停机           |

### 8.2 状态机

```
         启动 (10%)
          ↓
  ┌──→ 运行 ←──────────────────┐
  │     │ │                     │
  │     │ └─ 故障 (2%) ─→ 维护 ─┘
  │     │                  (修复后)
  │     └─ 换班 (按时间) ─→ 待机
  │                          │
  └──── 启动 ←───────────────┘
```

括号内为每分钟转移概率（运行态下），确保大部分时间设备在运行。

## 十、风险与应对

| 风险             | 概率  | 影响           | 应对                              |
| -------------- | --- | ------------ | ------------------------------- |
| 14 张图片旋转效果不够平滑 | 中   | 用户体验差        | 增加中间角度图片（拍摄或 AI 生成）；或降低旋转灵敏度    |
| 工厂侧无法提供数据接口    | 高   | Phase 2 无法推进 | Phase 1 做好 Mock，先展示价值；并行推动工厂侧评估 |
| 设备型号差异大        | 低   | 测点不通用        | 测点配置化，不硬编码                      |
| 浏览器 3D 性能问题    | 低   | 图片方案性能开销小    | 图片预加载 + 纹理压缩                    |
| 网络隔离（工厂内网）     | 中   | 部署受限         | Docker 离线部署包；或工厂内网部署            |
