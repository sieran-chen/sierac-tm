## 二、文件格式（按优先级排列）

我们的前端技术栈是 Three.js（浏览器 3D 引擎），最终使用的格式是 **glTF/GLB**。请按以下优先级提供：

### 首选：直接提供 glTF / GLB

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| **glTF** | `.gltf` + `.bin` + 贴图文件 | 开放标准，"3D 界的 JPEG"，我们可直接使用 |
| **GLB** | `.glb` | glTF 的二进制打包版，单文件，最方便 |

如果贵方的 CAD 软件支持直接导出 glTF/GLB，这是最省事的方式，我们无需做格式转换。

### 次选：提供中性 CAD 格式（我们自行转换）

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| **STEP** | `.step` / `.stp` | **优先推荐**，中性 CAD 格式，行业通用，转换兼容性最好 |
| **IGES** | `.igs` / `.iges` | 较老的中性格式，也可以转换，但不如 STEP 顺手 |

我们拿到 STEP/IGES 后，会用 Blender 或 CAD Exchanger 转换为 glTF。

### 也可接受：其他 3D 格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| **FBX** | `.fbx` | 游戏/影视行业常用，可转 glTF |
| **OBJ** | `.obj` + `.mtl` | 通用网格格式，可转 glTF，但不含动画和层级信息 |
| **STL** | `.stl` | 3D 打印格式，仅几何体，无颜色/材质，不推荐 |

### 不建议：原生 CAD 格式

如果只能提供 SolidWorks（`.sldprt` / `.sldasm`）、Creo（`.prt`）、Inventor（`.ipt`）等原生格式，请贵方先导出为上述任一格式再提供，因为原生格式需要对应的商业软件才能打开。

## 三、模型要求

### 3.1 部件拆分（关键需求）

**请将模型按功能部件分组/命名**，而不是一整个不可分割的几何体。这是实现「点击部件查看参数」和「告警部件高亮」的前提。

建议的部件拆分：

| 部件名称 | 英文命名建议 | 说明 |
|----------|-------------|------|
| 机架/框架 | `frame` | 设备主体框架 |
| 灌装头组件 | `filling_heads` | 所有灌装头（如果能单独拆分每个灌装头更好：`filling_head_01` ~ `filling_head_36`） |
| 灌装阀 | `filling_valves` | 灌装阀组 |
| 旋转台/转盘 | `turntable` | 中心旋转机构 |
| 进瓶传送带 | `conveyor_in` | 进瓶输送段 |
| 出瓶传送带 | `conveyor_out` | 出瓶输送段 |
| 进瓶星轮 | `star_wheel_in` | 进瓶星轮 |
| 出瓶星轮 | `star_wheel_out` | 出瓶星轮 |
| 液缸/料缸 | `tank` | 灌装液料缸 |
| 控制柜/电控箱 | `control_panel` | 电气控制柜 |
| 防护罩/外壳 | `housing` | 设备外壳/防护门 |
| 驱动电机 | `motor` | 主驱动电机 |
| 管路系统 | `piping` | 物料/气体管路 |
| CIP 清洗接口 | `cip_connection` | CIP 清洗接口（如有） |

> **最低要求**：至少将灌装头、传送带、控制柜、液缸作为独立部件。  
> **理想情况**：每个灌装头单独命名（`filling_head_01` ~ `filling_head_36`），这样可以实现单个灌装头的告警高亮。

### 3.2 精度与面数

| 项目 | 建议 |
|------|------|
| 总面数 | 50 万 ~ 200 万三角面（浏览器渲染性能考虑） |
| 细节程度 | 外观可辨识即可，不需要螺丝/螺母级别的细节 |
| 内部结构 | 不需要（我们只展示外观，不做剖面） |
| 尺寸单位 | 毫米（mm）或米（m），请注明使用的单位 |

> 如果原始 CAD 面数过高（如超过 500 万面），请在导出时做减面处理，或告知我们原始面数，我们可以在转换时减面。

### 3.3 材质与颜色

| 项目 | 建议 |
|------|------|
| 颜色 | 保留设备实际颜色（不锈钢银色、绿色护板等） |
| 材质贴图 | 有则更好（金属质感、玻璃透明等），没有也可以，我们可以后期在 Blender 中补 |
| 透明部件 | 如果有玻璃防护门等透明部件，请标注或设置透明材质 |

### 3.4 坐标系

| 项目 | 建议 |
|------|------|
| 原点 | 设备底部中心 |
| Y 轴 | 向上（Three.js 默认 Y-up） |
| 朝向 | 设备正面朝 +Z 方向 |

## 五、常见问题

**Q1：我们没有 glTF 导出功能，怎么办？**  
A：请导出 STEP 格式（`.step` / `.stp`），这是最通用的中性 CAD 格式，我们可以用工具转换。

**Q2：模型文件很大（几百 MB），正常吗？**  
A：CAD 原始文件可能很大，但转换为 glTF 后通常在 10-50 MB。如果 STEP 文件超过 200 MB，请先做减面处理或提供简化版。

**Q3：能否只提供外壳，不提供内部结构？**  
A：完全可以。我们只需要外观可视化，不需要内部零件。只提供外壳 + 关键部件（灌装头、传送带等）即可。

**Q4：部件拆分很麻烦，能否给整体模型？**  
A：整体模型也可以接受，但会影响「点击部件」和「告警高亮」功能。**最低要求是灌装头组件、传送带、控制柜作为独立部件**。

**Q5：需要动画吗？**  
A：本期不需要。后续如果要做旋转台转动、传送带运行等动画，会另行沟通。
