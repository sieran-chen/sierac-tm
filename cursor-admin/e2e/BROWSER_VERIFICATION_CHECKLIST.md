# 浏览器验证清单（部署后）

部署完成后在 **http://8.130.50.168:3000** 进行人工验证时可按本清单逐项勾选。

## 1. 用量总览
- [ ] 页面打开正常，左侧导航可见
- [ ] 成员筛选、日期范围可切换
- [ ] 每日请求趋势图与按成员汇总表有数据或空状态文案合理

## 2. 项目管理
- [ ] 进入「项目」页，列表与刷新按钮正常
- [ ] 点击「新建项目」：弹窗内可见「关联已有仓库」「自动创建（GitLab）」「自动创建（GitHub）」
- [ ] 选择「关联已有仓库」：有「已有仓库地址」输入与说明文案
- [ ] 保存/取消后弹窗关闭

## 3. 项目参与
- [ ] 从导航进入「项目参与」，URL 含 `/workspace`
- [ ] 有「按项目与成员汇总」或未归属说明；无 Hook 时可出现「尚未检测到 Hook 上报」横幅

## 4. 项目详情（若已有项目）
- [ ] 进入某项目详情（如 `/projects/1`）
- [ ] 成本/参与成员/Git 贡献 区域有「来自 Hook 上报」「来自 Git 采集」等数据来源标签
- [ ] 基本信息含工作目录规则、关联仓库等

## 5. 排行榜 / 我的贡献 / 激励规则
- [ ] 「排行榜」：周期选择与列表可打开
- [ ] 「我的贡献」：成员选择与得分展示可打开
- [ ] 「激励规则」：规则列表与重新计算入口可打开

## 6. 支出管理 / 告警
- [ ] 「支出管理」：列表或空状态/错误说明正常
- [ ] 「告警配置」「告警历史」：列表加载正常

## 自动化 E2E（可选）

对部署环境跑 Playwright：

```bash
cd cursor-admin/e2e
E2E_BASE_URL=http://8.130.50.168:3000 npx playwright test specs/spec-e2e-verification.spec.ts
```

若失败可查看 `test-results/` 或增加超时/重试后重跑。
